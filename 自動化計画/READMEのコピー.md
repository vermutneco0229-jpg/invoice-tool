# URURIA 発注管理システム ― 導入手順

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `Config.gs` | 定数・商品マスタ・代理店マスタ |
| `Utils.gs` | 共通ユーティリティ関数 |
| `Setup.gs` | 初期セットアップ（一度だけ実行） |
| `Menu.gs` | カスタムメニュー・onEdit ハンドラ |
| `FormSubmit.gs` | フォーム送信時の自動処理 |
| `Invoice.gs` | 請求書PDF生成 |
| `DeliveryNote.gs` | 納品書PDF生成 |

---

## 導入手順（初回のみ）

### Step 1: スプレッドシートを新規作成

1. [Google スプレッドシート](https://sheets.google.com) で新規スプレッドシートを作成
2. 名前を「URURIA発注管理」などに変更

### Step 2: GASエディタにコードを貼り付け

1. スプレッドシートのメニュー →「拡張機能」→「Apps Script」を開く
2. 左側のファイル一覧で `コード.gs` を選択
3. 以下の7ファイルを**それぞれ新しいファイルとして追加**してコードを貼り付ける
   - 左側の「＋」ボタン →「スクリプト」で新規ファイルを追加
   - ファイル名と内容を合わせる

| 追加するファイル名      | 内容                  |
| -------------- | ------------------- |
| `Config`       | Config.gs の内容       |
| `Utils`        | Utils.gs の内容        |
| `Setup`        | Setup.gs の内容        |
| `Menu`         | Menu.gs の内容         |
| `FormSubmit`   | FormSubmit.gs の内容   |
| `Invoice`      | Invoice.gs の内容      |
| `DeliveryNote` | DeliveryNote.gs の内容 |

> デフォルトの `コード.gs` は空にしておいて構いません。

### Step 3: 初期セットアップを実行

1. スプレッドシートに戻り、ページをリロード（F5）
2. メニューバーに「📦 発注管理」が表示される
3. 「📦 発注管理」→「① 初期セットアップ（初回のみ）」をクリック
4. 権限の許可を求めるダイアログが出るので「許可」をクリック
5. 「続けますか？」→「はい」をクリック
6. 1〜2分待つとセットアップ完了のダイアログが表示される

> **注意:** 初回実行時に「このアプリは確認されていません」と表示される場合は、
> 「詳細」→「安全でないページに移動（unsafe）」をクリックして許可してください。

### Step 4: ロゴ・印影を追加（手動）

1. Googleドライブ →「URURIA発注管理」フォルダ →「テンプレート」フォルダを開く
2. 「請求書テンプレート」をダブルクリックして開く
3. `【ここにロゴ画像を配置してください】` という部分の文字を削除して、ロゴ画像を挿入する
4. 同様に印影画像も挿入する
5. 「納品書テンプレート」も同じ手順で編集する

> **ヒント:** 画像の挿入は「挿入」→「画像」→「パソコンからアップロード」

### Step 5: 送料マスタを入力

1. スプレッドシートの「送料マスタ」シートを開く
2. 初期値（S〜なし）が入っているので、実際の送料に合わせて金額を修正する
3. 新しい区分が必要な場合は行を追加する

### Step 6: onEdit トリガーを設定

送料区分を選んだとき金額を自動再計算するために、追加でトリガーを設定します。

1. Apps Script エディタで `FormSubmit.gs` の `setupEditTrigger` 関数を選択
2. 「実行」ボタンをクリック
3. 権限を許可する

> または「📦 発注管理」メニューに追加することも可能です。

### Step 7: テスト動作確認

1. フォームURLを開いてテスト注文を送信する
2. スプレッドシートの「注文データ」シートにデータが自動追加されることを確認
3. 「送料区分」列のドロップダウンから送料を選択
4. 「📦 発注管理」→「② 選択行の請求書を生成」をクリック
5. GoogleドライブにPDFが生成されることを確認

## 日常の使い方

### 注文が来たとき（自動）

- フォームに回答が来ると、スプレッドシートに自動追加される
- 金額計算・入力チェックが自動実行される
- 「入力チェック」列が `OK` なら問題なし

### 請求書を発行するとき

1. 「注文データ」シートで対象行をクリック
2. 「送料区分」ドロップダウンで送料を選択（自動で合計が再計算される）
3. メニュー「📦 発注管理」→「② 選択行の請求書を生成」
4. 完了ダイアログのURLをコピーしてLINE/メールで送付
5. 送付後、「ステータス」列を「送付済」に変更

### 入金確認後

1. 「ステータス」列を「入金済」に変更
2. メニュー「📦 発注管理」→「③ 選択行の納品書を生成」
3. 完了ダイアログのURLをコピーして物流グループLINEに送付
4. ステータスが自動で「納品書発行済」に更新される

---

## ステータスの流れ

```
未処理 → 請求書発行済 → 送付済 → 入金済 → 納品書発行済
```

## カスタマイズ方法

### 商品を追加・変更する

`Config.gs` の `PRODUCTS` 配列を編集してください。
変更後は初期セットアップを再実行するか、フォームを手動で更新してください。

### 代理店を追加・変更する

`Config.gs` の `AGENCIES` 配列を編集するか、
スプレッドシートの「代理店マスタ」シートを編集してください。
フォームのドロップダウンを更新するには、Apps Script からフォームを再生成する必要があります。

### 送料パターンを変更する

スプレッドシートの「送料マスタ」シートを直接編集してください。
変更後、「📦 発注管理」→「送料ドロップダウンを更新」をクリックしてください。

## トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| 「📦 発注管理」メニューが表示されない | ページをリロード（F5）する |
| 請求書が生成されない | 設定シートのテンプレートIDとフォルダIDが空でないか確認 |
| 入力チェック列が更新されない | フォーム送信トリガーが設定されているか確認 |
| 送料ドロップダウンが表示されない | 「送料ドロップダウンを更新」を実行 |
| 「権限が必要」エラー | Apps Script エディタから権限を再許可する |

## システム概要

- **月額コスト:** 0円（Google Apps Script + Googleドライブ）
- **PDF保存先:** Googleドライブ「URURIA発注管理 > 請求書（または納品書） > YYYY年MM月」
- **テンプレート:** Googleドキュメント形式（ロゴ・印影は手動で配置）
- **送料計算:** 商品小計は課税対象、送料は非課税として計算
